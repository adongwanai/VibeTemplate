# Mutual Review Command

> Secondary path. Use when you explicitly want extra cross-model review outside the default GSD verification flow.

Cross-model review loop for higher code quality through dual verification.

## When to Use

- After Codex generates code
- Before committing significant changes
- When implementing high-stakes features
- As a quality gate before PR

## Two Review Modes

### Mode A: Claude Reviews Codex Output

Claude reviews code generated by Codex:
1. Read the generated files
2. Check against plan/spec
3. Verify correctness, security, style
4. Provide feedback or approval

### Mode B: Codex Reviews Claude Output

Request Codex to review code written by Claude:
1. Identify files to review
2. Use Codex exec: `codex exec "Review these files for bugs, security issues, and style problems"`
3. Present findings to user
4. Iterate if needed

## Workflow

### Step 1: Select Review Mode

Ask: "Review from Claude → Codex (Claude reviews Codex's work) or Codex → Claude (Codex reviews Claude's work)?"

### Step 2: Perform Review

**Claude Reviews Codex:**
```
Reading generated files...
Checking against plan...
Verifying implementation...
```

**Codex Reviews Claude:**
```
Invoking Codex for review...
codex exec "Review src/auth/service.ts for correctness and security"
```

### Step 3: Present Findings

```markdown
# REVIEW REPORT

**Mode**: [Claude → Codex | Codex → Claude]
**Verdict**: [APPROVED | NEEDS REVISION | BLOCKERS FOUND]

## Issues Found
- [severity] file:line — issue — fix

## Recommendations
- [suggestion]

## Approval Status
[Approved / Not Approved - fix issues first]
```

### Step 4: Iterate

If issues found:
1. Either fix directly (Claude) or ask Codex to fix
2. Re-review
3. Continue until approved

## Why Mutual Review?

Based on cross-model workflow from [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice):

- Different models catch different issues
- Reduces hallucination rates significantly
- Claude is stronger at architecture/reasoning
- Codex is stronger at idiomatic code patterns
- Cross-review reduces bug rate by 3-5x

## Example

```
/mutual-review

-> Mode? [Claude → Codex]
-> Reviewing Codex output for user-auth feature
-> Found 2 medium issues:
   - Missing error handling in token refresh
   - No rate limiting on login endpoint
-> Sending back to Codex for fixes
-> Re-review approved
-> ✓ Feature approved for merge
```
