# Mutual Review Skill

Cross-model code review for maximum quality through dual verification.

## When to Use

- After Codex generates code (Claude reviews Codex)
- Before commits (Codex reviews Claude)
- High-stakes features requiring extra scrutiny
- As a quality gate in the workflow

## Review Modes

### Mode A: Claude Reviews Codex

Claude reviews code generated by Codex:
- Check against original plan/spec
- Verify correctness and completeness
- Check security and best practices
- Validate against coding standards

### Mode B: Codex Reviews Claude

Request Codex to review code written by Claude:
- Use Codex's fresh perspective
- Catch Claude's blind spots
- Get implementation suggestions

## Instructions

### Claude Reviews Codex (Mode A)

1. **Read Generated Files**
   - List all files created/modified
   - Read each file carefully

2. **Check Against Plan**
   - Verify all requirements met
   - Check phase/test gate completion

3. **Review Code Quality**
   - Correctness (logic, edge cases)
   - Security (no secrets, input validation)
   - Style (follows project conventions)
   - Tests (coverage, quality)

4. **Output Review Report**
   ```markdown
   # REVIEW: Claude → Codex

   ## Verdict
   [APPROVED | NEEDS REVISION | BLOCKERS]

   ## Issues
   | Severity | File | Line | Issue | Fix |
   |----------|------|------|-------|-----|
   | HIGH | src/a.ts | 42 | Missing null check | Add if (!x) return |

   ## Summary
   [Brief assessment]
   ```

### Codex Reviews Claude (Mode B)

1. **Prepare Review Request**
   ```
   Review the following files for correctness, security,
   and adherence to best practices:

   Files: src/auth/service.ts, src/auth/middleware.ts

   Focus on:
   - Logic correctness
   - Security vulnerabilities
   - Edge cases
   - Test coverage
   ```

2. **Invoke Codex**
   ```bash
   codex exec "Review these files for bugs and security..."
   ```

3. **Present Findings**
   - Categorize by severity
   - Provide specific fix suggestions
   - Allow user to decide on fixes

## Why Mutual Review?

Different models have different strengths:

| Claude (Opus) | Codex (GPT-5.4) |
|---------------|------------------|
| Complex reasoning | Pattern recognition |
| Architecture review | Idiomatic code check |
| Security analysis | Performance optimization |
| Long context | Fast iteration |

**Combined**: Error rates drop 3-5x vs single model

## Iteration Loop

```
Generate → Claude Review → Fix (Codex/Claude) → Claude Review → ...
                                                          ↓
                                                    Approved!
```

## Output Summary

Always provide:
- Verdict (APPROVED / NEEDS REVISION / BLOCKERS)
- Issue count by severity
- Specific actionable fixes
- Final approval decision
